# Scalar / Generic Web Component

If you are using [Scalar](https://scalar.com/), plain HTML/JS, or any other framework, you can use our generic Web Component.

## Installation

```bash
npm install vetradocs-scalar
```

**[View on npm â†’](https://www.npmjs.com/package/vetradocs-scalar)**

## Backend Setup
 
You strictly need a backend service to handle AI requests.
 
1.  Run the backend creator:
    ```bash
    npx create-vetradocs-backend@latest
    ```
2.  Choose **Cloudflare Workers** (Recommended) or Node.js.
3.  Deploy and get your URL and API Key.
 
See [Backend Setup](./backend-setup) for a step-by-step guide.
 
Update your widget attributes with the backend URL and API Key:
 
```html
<vetradocs-widget 
    api-endpoint="https://your-backend.com"
    api-key="your-secret-api-key"
></vetradocs-widget>
```
 
## Quick Start

### 1. Add Script

Load the script in your HTML file (e.g., `index.html`):

```html
<script type="module" src="./node_modules/vetradocs-scalar/dist/vetradocs-scalar.js"></script>
```

Or via CDN (once available via unpkg/jsdelivr).

### 2. Add Component

Add the custom element to your page structure:

```html
<vetradocs-widget 
    api-endpoint="/api/chat"
    index-path="/search-index.json"
    title="Scalar AI"
    accent-color="#8e4ec6"
></vetradocs-widget>
```

### 3. Generate Index

If you have Markdown files, use the included build script:

```bash
npx vetradocs-build-generic --docs ./docs --output ./public/search-index.json
```

If you are using Scalar with an OpenAPI spec, you will need to create a custom indexer to feed your API spec into the Orama search database, or manually create a markdown representation of your API for indexing.

## Configuration Attributes

| Attribute | Default | Description |
|-----------|---------|-------------|
| `api-endpoint` | `/api/chat` | Chat API URL |
| `index-path` | `/search-index.json` | Path to search index |
| `title` | `AI Assistant` | Chat window title |
| `accent-color` | `#f97316` | Main theme color |
| `placeholder` | `Ask a question...` | Input placeholder |
| `position` | `bottom-right` | `bottom-right`, `bottom-left`, `bottom-center` |
